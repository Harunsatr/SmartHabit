NEXTAUTH_SECRET not set\n2. Session cookie corrupted\n3. Database connection failed\n\n**Solution**:\n```bash\n# 1. Clear cookies\n# DevTools → Application → Cookies → Clear all\n\n# 2. Verify NEXTAUTH_SECRET\necho $NEXTAUTH_SECRET\n\n# 3. Test database connection\nnpx prisma studio\n\n# 4. Check logs\nnpm run dev  # Look for errors\n```\n\n### Issue: \"Unauthorized\" at `/admin`\n**Symptoms**: Redirects to /dashboard even when logged in as admin\n\n**Causes**:\n1. User role not set to \"ADMIN\" in database\n2. Session callback not adding role\n3. JWT token corrupted\n\n**Solution**:\n```typescript\n// 1. Check database\nnpx prisma studio\n// Verify User.role = \"ADMIN\"\n\n// 2. Check session in browser\nfetch('/api/auth/session')\n  .then(r => r.json())\n  .then(d => console.log('Role:', d.user.role))\n// Should show: Role: ADMIN\n\n// 3. Check JWT in cookies\n// DevTools → Application → Cookies\n// Copy next-auth token and decode at jwt.io\n```\n\n### Issue: Session Not Persisting After Refresh\n**Symptoms**: Have to re-login after page refresh\n\n**Causes**:\n1. Secure cookie flag wrong\n2. SameSite policy issue\n3. Session maxAge exceeded\n\n**Solution**:\n```bash\n# Check auth.config.ts settings\n# useSecureCookies should be true in production\n# sameSite should be 'lax'\n# maxAge should be 7 * 24 * 60 * 60\n\n# Verify cookie in DevTools\n# Check Expires/Max-Age value\n```\n\n### Issue: CORS Error When Fetching Session\n**Symptoms**: 404 or CORS error when trying to fetch /api/auth/session\n\n**Solution**:\n```typescript\n// Use correct endpoint\nfetch('/api/auth/session')  // ✅ Correct\nfetch('/auth/session')       // ❌ Wrong\n\n// Ensure request includes credentials\nfetch('/api/auth/session', {\n  method: 'GET',\n  credentials: 'include',  // Include cookies\n})\n```\n\n---\n\n## 📊 Monitoring & Maintenance\n\n### Logs to Monitor\n```\n✓ [Session] User session logs (development)\n✓ [Middleware] Non-admin access warnings\n✓ [guardAuth] API auth failures\n✓ NextAuth errors in console\n```\n\n### Regular Checks\n```bash\n# Weekly\n- Check error logs\n- Verify admin users exist\n- Test login flows\n\n# Monthly  \n- Review failed login attempts\n- Check for suspicious activities\n- Verify HTTPS/SSL certificate\n\n# Quarterly\n- Update dependencies\n- Review security settings\n- Test disaster recovery\n```\n\n### Health Checks\n```bash\n# Is auth working?\ncurl -X GET https://yourdomain.com/api/auth/session \\\n  -H \"Cookie: next-auth.session-token=YOUR_TOKEN\"\n\n# Can admin login?\n# Manually test in browser\n\n# Database working?\nnpx prisma db execute --stdin < \"SELECT COUNT(*) FROM \\\"User\\\";\"\n```\n\n---\n\n## 🔄 Updating Auth System\n\n### If Changing Password Hashing\n1.
